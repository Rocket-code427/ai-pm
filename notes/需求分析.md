# 会议纪要也要支持上传文字文档的。还有需求文档的上传及AI生成，需要明确逻辑。

## 会议纪要：上传文字文档

- 前端：会议纪要区域 → 增加「📄 上传文字」按钮（上传txt/md/pdf/docx）
- 后端：新增 POST /api/projects/{project_id}/ai/meeting-from-text
- 逻辑：提取文件文本 → 调用LLM生成结构化纪要 → 保存到 meetings/

## 需求文档：上传文字→AI解析

- 前端：需求文档区域 → 增加「📝 文字文档→AI解析」按钮
- 后端：新增 POST /api/projects/{project_id}/ai/requirement-from-text
- 逻辑：提取文件文本 → 调用LLM生成结构化需求 → 保存到 requirements/

## 统一逻辑

| 输入 | 功能 | 输出位置 |
|------|------|----------|
| 录音文件 | 转录+结构化 | meetings/ + 可选转requirements/ |
| 文字文档 | 直接结构化 | meetings/ + 可选转requirements/ |
| 文字文档 | AI解析为需求 | requirements/ |
| 会议纪要 | 自动转需求 | requirements/ |
| 需求文件 | 生成PRD | prd/ |

## 前端按钮布局

### 会议纪要区域（输入层）
- 上传录音（已有）
- **上传文字**（新增）

### 需求文档区域（工作流）
- 上传（已有，手动上传需求文件）
- **文字文档→AI解析**（新增，上传文档自动转需求）
- AI生成（已有，从会议纪要生成）

### PRD区域（工作流）
- 从需求生成（已有）
- 访谈生成（已有）
- 文档直传（已有）
